## Epic 1 – Configuration, security & personalization

**Goal:** Have a safe and personalized experience.

1. **US-001 – Multi-currency & localization (future)**

    - _As a_ user
    - _I want_ the app to support my local currency (COP) and possibly others in the future
    - _So that_ I can see all numbers in a consistent way.
    - **Acceptance criteria**

      - Default currency is COP.
      - Currency is shown consistently in all views.
      - (Future) Ability to add accounts in other currencies and show converted totals.

2. **US-002 – Authentication & data privacy**

    - _As a_ user
    - _I want_ secure login and my financial data protected
    - _So that_ I feel safe using the app.
    - **Acceptance criteria**

      - Secure login (email + password, or SSO later).
      - Basic session management (auto-logout after inactivity, etc.).
      - Sensitive data is never shown without being logged in.

---

## Epic 2 – Unified view of accounts & balances

**Goal:** See my full financial picture in one place.

3. **US-003 – Connect and model all my accounts**

   - _As a_ user
   - _I want_ to register all my financial accounts (Bancolombia, Nubank, Nequi, RappiAccount, RappiCard, cash, loans, etc.)
   - _So that_ the app can show a complete picture of my finances in one place.
   - **Acceptance criteria**

     - I can create an account with: name, type (checking, savings, credit card, loan, “money people owe me”, etc.), currency, and initial balance.
     - I can edit or archive accounts.
     - Archived accounts no longer accept new transactions but remain visible historically.

4. **US-004 – Global balance overview**

   - _As a_ user
   - _I want_ to see a dashboard with my total assets, total debts, and net worth
   - _So that_ I can quickly understand my financial situation.
   - **Acceptance criteria**

     - The dashboard shows:

       - Total cash & liquid assets
       - Total debt (credit cards + loans I owe)
       - Total “receivables” (money others owe me)
       - Net worth = assets – debts

     - I can see a breakdown by account.

5. **US-005 – Per-account balance view**

   - _As a_ user
   - _I want_ to open any account and see its current balance and recent movements
   - _So that_ I know exactly how much money is there at any time.
   - **Acceptance criteria**

     - Account detail page shows current balance.
     - Shows a list of last N transactions.
     - Shows filters by date range and transaction type

## Epic 3 – Unified transaction feed

**Goal:** See all money movements across all accounts in one list.

6. **US-006 – Global transaction feed**

   - _As a_ user
   - _I want_ to see a single list of all my transactions across all accounts
   - _So that_ I can understand where my money is coming from and going to.
   - **Acceptance criteria**

     - List includes: date, description, account, category, amount.
     - I can filter by:

       - Account(s)
       - Date range
       - Category (e.g., groceries, rent, loan repayment, etc.)
       - Type (income, expense, transfer)
       - Description
       - Amount

     - The filter is done by a single input text and system is able to smartly understand how to search and filter.

7. **US-007 – Manual transaction creation**

   - _As a_ user
   - _I want_ to manually add, edit, and delete transactions
   - _So that_ I can fix errors and register movements that are not imported automatically.
   - **Acceptance criteria**

     - I can create a new transaction with: date, account, amount, type (income/expense/transfer), description, category.
     - Editing a transaction updates the account balance accordingly.
     - Deleting a transaction updates the account balance accordingly.

8. **US-008 – Internal transfers between accounts**

   - _As a_ user
   - _I want_ to record transfers between my own accounts (e.g., Bancolombia → Nubank, Bancolombia → RappiAccount)
   - _So that_ the money is not counted twice as income/expense.
   - **Acceptance criteria**

     - I can create a “transfer” that affects a source and destination account.
     - The UI prevents double-counting (not treated as new income or new expense).
     - The transaction appears in both accounts, linked as a transfer.

## Epic 4 – Bank statement upload & AI extraction

**Goal:** Automate transaction creation by parsing statements.

9. **US-009 – Upload a bank statement file**

    - _As a_ user
    - _I want_ to upload my Bancolombia/Nubank/Nequi/RappiAccount statements (PDF, CSV, etc.)
    - _So that_ the system can extract the transactions automatically.
    - **Acceptance criteria**

      - I can select which account the file belongs to.
      - The app accepts at least one format (start with Bancolombia’s most common format).
      - The file is stored and its processing status is visible (pending / processed / failed).

10. **US-010 – AI-based transaction extraction**

    - _As a_ user
    - _I want_ the app to use AI to read the statement and extract transactions
    - _So that_ I don’t have to input everything manually.
    - **Acceptance criteria**

      - The app extracts at least: date, description, amount, type (debit/credit), and balance (if available).
      - The extracted transactions are shown in a “review” screen before being saved.
      - I can bulk-accept or individually edit transactions before confirming.

11. **US-011 – Duplicate detection and merge**

    - _As a_ user
    - _I want_ the system to detect duplicate transactions when I upload overlapping statements
    - _So that_ I don’t have the same movement twice.
    - **Acceptance criteria**

      - App warns about potential duplicates using rules (same account, date, amount, similar description).
      - I can choose to ignore, merge, or create anyway.

## Epic 5 – Savings & interest-bearing accounts (Nubank & others)

**Goal:** Track interest gains and true performance of savings.

12. **US-012 – Model interest-bearing accounts**

    - _As a_ user
    - _I want_ to configure accounts that earn interest like Nubank and Nubank CDT
    - _So that_ I can track how much interest I earn each month.
    - **Acceptance criteria**

      - I can mark an account as “interest-bearing”.
      - I can define: interest rate, frequency (daily/monthly), and whether interest is paid out or reinvested (capitalized).
      - The app can record interest as separate income transactions.

13. **US-013 – Track periodic contributions to savings**

    - _As a_ user
    - _I want_ to track how much money I send each month from Bancolombia to Nubank / RappiAccount
    - _So that_ I can see my saving behavior over time.
    - **Acceptance criteria**

      - Transfers from Bancolombia to Nubank are visible and tagged as “savings contributions”.
      - I can see historical graphs of contributions per month.

## Epic 6 – Credit card tracking (RappiCard)

**Goal:** Understand how much I owe, how much interest I pay, and avoid surprises.

14. **US-014 – Model credit cards with limits and billing cycles**

    - _As a_ user
    - _I want_ to set my RappiCard with its credit limit and billing cycle
    - _So that_ the app can correctly track my open balance and due dates.
    - **Acceptance criteria**

      - Fields: credit limit, closing date, due date, interest rate when balance is not fully paid.
      - The card is treated as a liability account.

15. **US-015 – Track credit card transactions**

    - _As a_ user
    - _I want_ to see all my RappiCard transactions in the app
    - _So that_ I can track how much I owe and what I spent on.
    - **Acceptance criteria**

      - Transactions show date, merchant/description, amount, category.
      - Transactions contribute to the current statement balance.

16. **US-016 – Track statement balance and interest**

    - _As a_ user
    - _I want_ to see how much I owe on my RappiCard each cycle and how much I pay in interest when I don’t pay full
    - _So that_ I understand the real cost of using my card.
    - **Acceptance criteria**

      - The app calculates:

        - Statement balance
        - Minimum payment (optional)
        - Interest charged if full payment is not made by due date.

      - If I pay the full statement within 1 month, interest = 0 (as per RappiCard behavior).
      - If I don’t pay in full, the app shows estimated or actual interest.

17. **US-017 – Record credit card payments**

    - _As a_ user
    - _I want_ to record payments from my bank accounts to my credit card
    - _So that_ both accounts reflect the real balances.
    - **Acceptance criteria**

      - I can select a bank account (e.g., Bancolombia) as payment source.
      - The app creates:

        - An expense (or transfer) transaction in the bank account.
        - A payment transaction in the credit card account, reducing the balance.

## Epic 7 – Loans I’ve given (money people owe me)

**Goal:** Track who owes me money, balances, and interests.

18. **US-018 – Create loans I’ve given**

   - _As a_ user
   - _I want_ to create a loan for each person I’ve lent money to (Adriana, Cristian, my mom, my aunt, my cousin, María Alejandra)
   - _So that_ I can track how much each one owes me.
   - **Acceptance criteria**

     - Loan fields: borrower name, principal amount, start date, interest configuration (none, fixed rate, dynamic/variable).
     - Loans with no interest should still track principal and repayments.
     - Each loan appears as an “asset” in my overview (money people owe me).

19. **US-019 – Configure fixed interest rate loans**

   - _As a_ user
   - _I want_ to set a fixed interest rate for loans like Cristian’s
   - _So that_ the app can calculate interests owed over time.
   - **Acceptance criteria**

     - I can set: interest rate (e.g., X% monthly or annually), compounding frequency (if applicable).
     - The system calculates accrued interest over time and shows:

       - Principal remaining
       - Interest accrued
       - Total amount owed.

20. **US-020 – Support dynamic interest rate loans**

   - _As a_ user
   - _I want_ to configure loans with dynamic interest rates like Adriana’s
   - _So that_ I can adjust interest rules over time.
   - **Acceptance criteria**

     - I can update the interest rate (or formula) at specific dates.
     - History of changes is kept.
     - Accrued interest is recalculated based on the defined periods.

21. **US-021 – Record payments for loans**

    - _As a_ user
    - _I want_ to record payments against each loan
    - _So that_ I can track how much is left to pay.
    - **Acceptance criteria**

      - I can log a payment with: date, amount, from which bank account it was paid (e.g., Bancolombia), and whether it goes first to interests or principal.
      - The system updates the remaining balance (principal + interest).
      - The payment is also recorded as a transaction in the corresponding bank account.

22. **US-022 – Loan overview**

    - _As a_ user
    - _I want_ to see a list of all my loans with current status
    - _So that_ I know how much everyone owes me at a glance.
    - **Acceptance criteria**

      - For each loan I can see: borrower, principal, total paid, interest accrued, remaining balance, last payment date.
      - Ability to filter by active/fully repaid loans.

## Epic 8 – Insights & reporting

**Goal:** Understand patterns, not just raw data.

23. **US-023 – Category-based spending and income breakdown**

    - _As a_ user
    - _I want_ to see how much I spend and earn by category
    - _So that_ I can see where my money goes and adjust habits.
    - **Acceptance criteria**

      - I can see a summary by category per month.
      - Each category shows total amount and percentage of total spending/income.
      - I can filter by account or see the global view.

24. **US-024 – Timeline of net worth**

    - _As a_ user
    - _I want_ to see how my net worth changes over time
    - _So that_ I can evaluate my financial progress.
    - **Acceptance criteria**

      - Line chart of net worth over time (monthly at first).
      - Option to include/exclude specific accounts (e.g., exclude loans I’ve given, include only cash + savings).

25. **US-025 – Loan repayment schedules & status**

    - _As a_ user
    - _I want_ to see repayment progress for each loan I’ve given
    - _So that_ I know who is paying on time and who is delayed.
    - **Acceptance criteria**

      - For each loan I can see a timeline of payments.
      - The app highlights overdue payments if a schedule is defined.
      - Shows remaining term (if fixed schedule exists).

## Suggested MVP scope

If you want a lean v1, I’d prioritize:

- **Must-have for MVP**

  - US-003, US-004, US-005 (accounts + balances)
  - US-006, US-007, US-008 (transactions + transfers)
  - US-018, US-021, US-022 (loans basic, no fancy dynamic rate yet)
  - US-014, US-015, US-017 (basic credit card tracking + payments)
  - US-009, US-010 (statement upload + AI extraction basic)
  - US-002 (auth & basic security)

- **Nice-to-have after MVP**

  - Dynamic interest (US-020)
  - Detailed interest modeling for RappiCard (US-016)
  - Savings analytics (US-012, US-013)
  - Reports & net worth charts (US-023, US-024, US-025)
  - Duplicate detection (US-011)
